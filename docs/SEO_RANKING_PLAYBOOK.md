# SEO Ranking Playbook

This playbook explains how this system helps push rankings and what operators should do to maximize results.

## 1) Ranking Model Used by the Workflow

The system follows a practical SEO loop:

1. Intent mapping via keyword generation.
2. Content production around mapped intent.
3. Clean technical publishing (metadata, structure, indexing signals).
4. Performance measurement (impressions/clicks/position).
5. Iterative optimization cycles.

## 2) What the System Automates

1. Keyword generation with niche context.
2. Comparison keyword discovery ("X vs Y") for high-conversion queries.
3. Draft content generation from selected keywords.
4. Publish workflows (internal + external).
5. Structured output support (slug/meta/schema handling).
6. Metrics capture (manual, simulated, and GSC).

## 3) What Actually Improves Rankings

1. Correct search intent targeting.
2. Consistent publishing velocity.
3. Strong topical depth for each niche.
4. Better CTR via title/meta iteration.
5. Content refresh on decaying URLs.
6. Distribution and internal linking from main site.

Automation helps speed. Ranking still depends on quality + relevance + authority.

## 4) Keyword Strategy by Funnel Stage

1. TOFU (awareness):
- Use informational long-tail queries.
- Target broader educational topics.

2. MOFU (consideration):
- Use "best", "alternatives", "how to choose" terms.
- Build comparison clusters.

3. BOFU (decision):
- Use explicit product/service comparisons and commercial modifiers.
- Prioritize conversion-intent pages.

Recommended mix per 10 new keywords:
- 5 informational
- 3 comparison
- 2 transactional/commercial

## 5) Publish Cadence Recommendation

1. New websites:
- 3 to 5 posts per week for first 6 to 8 weeks.

2. Mature websites:
- 1 to 3 new posts per week + refresh old winners.

3. Refresh policy:
- Revisit pages every 30 to 45 days if CTR or position declines.

## 6) CTR Optimization Loop

For pages with impressions but low clicks:

1. Rewrite title for clarity and benefit.
2. Improve meta description for stronger action.
3. Ensure keyword appears naturally in title + intro.
4. Re-publish updates and track next 14 days.

## 7) Position Recovery Loop

For pages stuck in positions 8 to 25:

1. Expand sections to better match search intent.
2. Add FAQ and comparison blocks.
3. Add internal links from relevant published pages.
4. Add external references where useful.
5. Re-index and monitor trend.

## 8) Technical SEO Controls in This System

1. Sitemap endpoint support.
2. Robots rules support.
3. JSON-LD support.
4. Clean slug generation.
5. Meta description handling.

These controls reduce crawl/index friction and improve discoverability.

## 9) Metrics to Watch Weekly

1. Impressions trend by website.
2. Click growth and CTR trend.
3. Average position movement.
4. Published posts count by niche.
5. Top 10 improving pages and top 10 declining pages.

## 10) 90-Day Execution Blueprint

1. Days 1 to 14:
- Setup tenant/website/auth.
- Publish initial cluster set.
- Validate indexing and analytics.

2. Days 15 to 45:
- Increase publishing cadence.
- Add comparison clusters.
- Start CTR optimization on early pages.

3. Days 46 to 90:
- Scale highest-performing niche clusters.
- Refresh declining pages.
- Push internal-linking improvements.
- Use metrics to remove low-value topics.

## 11) Important Reality Check

No tool can guarantee ranking for every keyword. This system provides:

1. faster production,
2. cleaner SEO structure,
3. better measurement,
4. repeatable optimization.

Final ranking outcome depends on competition, domain authority, quality, and distribution.
